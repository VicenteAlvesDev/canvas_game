
UPDATE LOG – Coletor Cidadão
Versão 3.0 - Dashing Forward

[ SISTEMA GERAL ]

- Tela de Carregamento: Adicionada tela de loading com barra de progresso animada.
- Modal de Nome: Novo modal para o jogador inserir seu nome, exibido acima do personagem.
- Estados do Jogo: Adicionado estado "loading" e melhoria na organização geral.
- Remoção de Bugs: Prevenção da rolagem da página ao usar teclas de controle.
- Versão atualizada para v3.0.

[ SISTEMA DE TELAS ]

- Tela Inicial (Start):
  - Reformulada completamente.
  - Menu com opções (INICIAR, PERSONALIZAÇÃO).
  - Sistema de fundo com parallax (Sky, Far, Mid, Front).
  - Melhorias visuais com gradientes e efeitos.

- Tela de Morte (Death):
  - Mantida e integrada ao novo sistema de estados.

[ ARTE E VISUAIS ]

- Fundo do Site: Imagem "spr_siteBG.png" com overlay semitransparente.
- Banner: Adicionado "spr_banner.png" acima do jogo.
- Seção História: Texto explicativo sobre o jogo.
- Seção Controles: Lista detalhada de comandos.
- Nome do Jogador: Implementação de drawName().
- Sistema de Dicas:
  - Frases ambientais ao reciclar.
  - Efeito de shake na tela.
  - Fade-out e acompanhamento do personagem.

[ ÁUDIO ]

- Novo efeito sonoro: sfxDash.mp3 para o dash.

[ JOGABILIDADE ]

- Habilidade de Dash:
  - Tecla Q (teclado) ou RT/R2 (gamepad).
  - Movimento rápido na direção atual.
  - Cooldown de 30 frames.

- Movimentação do Player aprimorada.
- Dicas aparecem a cada 5 reciclagens.

[ MAPA E BACKGROUND ]

- Sistema de Paralaxe com bgOffset.
- Novas imagens: bg_sky, bg_far, bg_mid, bg_front.
- Função drawLayer() para renderização.

[ INTERFACE DO JOGO (UI/UX) ]

- Layout HTML:
  - Estrutura com game-container, history e controls.
  - Rodapé com direitos autorais.

- Estilização:
  - Design moderno com fundo escuro transparente.
  - Bordas arredondadas e sombras.
  - Cor principal: #38bdf8.

- Novos elementos HTML:
  - #loading-screen, .modal, .control-group, etc.

[ SISTEMA DE DIFICULDADE ]

- Progressão mantida (velocidade aumenta a cada 5 reciclagens).
- Indicadores visíveis e funcionais.

[ CÓDIGO E ARQUITETURA ]

- Modularização:
  - loading_loop, start_loop, canvas_update, canvas_draw, end_loop.

- Utilitário:
  - Função approach() para animações suaves.

- Organização:
  - Variáveis agrupadas (game, menu, bgOffset, tips).

- Código mais limpo e preparado para expansão.

[ CONTROLES ]

- Novo mapeamento:
  - Q / RT → Dash
  - A/X → Reiniciar
  - B/◯ → Menu

- Teclas não afetam mais a rolagem da página.
